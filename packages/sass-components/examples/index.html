Callout 组件样式测试

这个页面展示了为 CalloutProcessor.ts 设计的各种 callout 样式。每种类型都有独特的颜色标识。

信息类型 (蓝色系)

信息提示

这是一个信息类型的 callout，用于展示重要信息。

data-callout="info"
笔记

这是一个笔记类型的 callout，用于记录要点。

data-callout="note"

摘要类型 (青蓝色系)

摘要

这是一个摘要类型的 callout，用于概括要点。

data-callout="abstract"
TLDR (太长不看)

这是一个 TLDR 类型的 callout，用于快速概要。

data-callout="tldr"

成功类型 (绿色系)

成功

这是一个成功类型的 callout，用于表示操作成功。

data-callout="success"
已完成

这是一个已完成类型的 callout，用于表示任务完成。

data-callout="done"

警告类型 (黄色系)

警告

这是一个警告类型的 callout，用于提醒注意事项。

data-callout="warning"
注意

这是一个注意类型的 callout，用于强调谨慎操作。

data-callout="caution"

错误/危险类型 (红色系)

危险

这是一个危险类型的 callout，用于警示严重风险。

data-callout="danger"
错误

这是一个错误类型的 callout，用于标识错误信息。

data-callout="error"
Bug

这是一个 bug 类型的 callout，用于标记程序错误。

data-callout="bug"

提示类型 (紫色系)

提示

这是一个提示类型的 callout，用于分享有用的技巧。

data-callout="tip"
重要

这是一个重要类型的 callout，用于突出关键信息。

data-callout="important"

问题类型 (橙色系)

问题

这是一个问题类型的 callout，用于提出疑问。

data-callout="question"
帮助

这是一个帮助类型的 callout，用于提供协助信息。

data-callout="help"

其他类型

引用

这是一个引用类型的 callout，用于引述他人观点。

data-callout="quote"
示例

这是一个示例类型的 callout，用于展示具体例子。

function example() {
    return "Hello, World!";
}
data-callout="example"
待办

这是一个待办类型的 callout，用于标记未完成的任务。

  • 任务1：完成设计稿
  • 任务2：编写代码
  • 任务3：测试功能
data-callout="todo"
